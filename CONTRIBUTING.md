# Contributing to Eventix

Thank you for considering contributing to Eventix. This document outlines the guidelines and process for contributing to the project.

## 1. Code of Conduct
By contributing to this project, you agree to uphold the standards of respectful, constructive, and inclusive communication. All interactions should be professional and supportive. Harassment, discrimination, or abusive behavior will not be tolerated.

## 2. How to Propose a Feature
If you have an idea for a new feature:
1. Open a new issue titled "Feature Proposal: <your feature name>".
2. Describe the problem the feature solves, the proposed solution, and possible alternatives.
3. Wait for maintainers to review and approve before starting any implementation.

## 3. How to File a Bug Report
To report a bug:
1. Open a new issue titled "Bug: <short description>".
2. Include reproduction steps, expected behavior, observed behavior, screenshots or logs if applicable, and environment details (OS, version, runtime).
3. Maintain clarity and completeness so the issue can be verified quickly.

## 4. Making a Contribution
Follow these steps when contributing code:
1. Fork the repository on GitHub.
2. Create a new branch from `main` with a descriptive name (for example, `fix/login-timeout` or `feature/calendar-widget`).
3. Write clean, documented, and tested code.
4. Ensure your changes pass all linting and tests.
5. Submit a pull request (PR) with a clear description of your changes, the motivation, and any related issue numbers.
6. Be responsive to review comments. Your PR will be merged once it meets the project standards.

## 5. Coding Standards
- Follow the existing code style and naming conventions in the project.
- Keep commits logically grouped and include meaningful commit messages.
- Write unit tests for new code when applicable.

## 6. Pull Request Requirements
A pull request must:
- Pass CI checks.
- Not introduce breaking changes unless approved in advance.
- Include documentation updates if the changes affect user-facing behavior.
- Reference related issues when applicable.

## 7. Project Setup
To run the project locally:
1. Clone the repository.
2. Install dependencies as described in the README.
3. Run the development server and tests to ensure your environment is working before contributing.

## 8. Licensing
By submitting a contribution, you agree that your work will be licensed under the same open-source license used by this project.

---

Thank you for contributing to Eventix and helping improve the project.
